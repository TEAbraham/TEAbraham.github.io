Caissa Heatmap

The Caissa Database is one of the largest databases of chess games free and open to the public. The criteria for each game is that at least one player has the title of master and the game has lasted more than five moves. It is a compilation of over four million games in portable game notation (PGN). I was able to clean and organize the data using Go language and the Google Cloud Platform for speed and efficiency. The following visualizations are both user friendly examples of how D3.js can be used to tell a story and abstract examples of how a neural network like AlphaZero might use reinforcement learning to train itself with and against certain tree positions.

The heatmaps to the right visualize such training. First, an example of how a single node might "think" about how each queen utilizes the board. Second, how a neural network of nodes might work together to disseminate how every piece utilizes the board in different positions, how often squares are moved to and from, how often they are used to capture opposing pieces, and even how often they are used to check the opposing king. 




The Small Opening

The visualization to the right is a simple rendering of the frequency of various chess openings in the Caissa Database.  A chess opening is the first moves or opening line of moves that each side makes to develop their pieces.  Here we have a D3 Sunburst with each arc representing a different move; darker arcs correspond to dark moves, and lighter arcs correspond to light moves.  Each color corresponds to a different first move. Hover over any arc to see its unique board position, line notation and title in the Encyclopedia of Chess Openings.  Use the buttons below to take a closer look at one of the four most common first moves by Grand Masters.

The Big Opening

This is a more complex and ideally useful rendering of the Caissa Opening.  The user can manipulate the graph or the board itself to learn about the frequency and utilization of every single opening.  The chessboard, sunburst, and breadcrumb trail of each variation updates simultaneously when a piece makes a legal move or a particular arc is clicked on.  The legend at the onset of the opening color codes each piece, so we can see how dynamic or violent a particular line or opening is in terms of material lost and gain.  We can also see what percentage of the database each position represents, as well as the notation and title just as before.

USGS Earthquakes Map

This visualization was a bootcamp assignment with experimental add-ons. The assignment was to load the most recent earthquake data from the USGS, and to plot the occurences with respect to their seismic intensity on the Richter scale.  The feature add-ons include a sliding scale to isolate occurences within a single period, and the color coding of the Richter scale.  The color coding was necessary because assigning radii based on distances using the Mercator Projection map seemed to distort the truth of the data.

USGS Observable Globe

The following is an attempt to correct the former distortion of USGS earthquake data utilizing an Observable Notebook. Observable is the next step in Mike Bostock's vision for D3.js, where the data scientist is able to run live code from an environment that incorporates endless javascript libraries, dataflows, and runtime to liberate the storyteller from their previous constraints.  Here we are capable of adjusting ever variable and project our data onto a globe to fix the distortion problem we previously had with the Mercator Projection.

Census Bureau and Behavioral Risks

The chart to the right is one of the most simple, elegant, and practical applications of data visualization using D3.  The dataset is a list of all fifty states and the District of Columbia, and the percentage of their respective populations facing certain health risks as well as various other demographic demarcations.  Use the tooltip to hover over each state to see that data associated with it.  Click different x and y axis labels to make your own comparisons and note trends, the data points and axis scales will update automatically.  The rest of the web page illustrates the most noteable trends and the stories we are able to tell given said information.

Chicago Air Quality Dashboard

This project is the brainchild of a team of developers that included financial analysts and chemical engineers.  It uses federal data and checkpoints to create a dashboard for Chicago air quality.  

The Real-time page of the dashboard uses Leaflet and incorporates GoogleApis to place the air monitor checkpoints as well as current traffic patterns.  The primary goal here is to observe how traffic pollutants affect air quality at any given hour of the day, using the slider to look at previous hours.  The current Air Quality Index (AQI) is given in contagion particles per million air particles.  

The Historical page of the dashboard is comprised of two charts.  The first chart is a map of Chicago with every monitoring location included to the left, and a radial graph resembling a clock above a period chart resembling a calendar to the right.  The radial graph includes every hour of the day so we can see pollution spikes on an average day, where the calendar chart acts almost like a heatmap allowing us to see the most polluted days in the past month. All charts and tabs are interactive so you can isolate a pollutant, monitoring station, or individual day.  The second chart is not unlike the previously utilized D3 scatterplot in that y axis labels for each pollutant can be clicked on and isolated.  However, in addition to these functions, the entire graph is scalable to observe minor and major trends based on time, simply use the scroll function on your mouse or trackpad.

State Historical Accuracy

The following is an experiment in visual storytelling using Tableau.  The opening tab lists every state and their history of participating in presidential elections.  The alternate tabs visualize the same information with heatmaps scaled towards the percentage of how often a particular state's electoral votes correctly correlate with the winner of the presidential election. We can see how the heatmaps can be deliberately misleading, providing no information of how old a state actually is or how many elections it has actually been included in.

MNIST Data and Image Classifier

The MNIST dataset is one of the most common datasets used for image classification and accessible from many different sources. In fact, even Tensorflow and Keras allow us to import and download the MNIST dataset directly from their API.
This is an initial build of a convolutional neural network to classify hand written digits with Tensorflow’s Keras API, achieving an accuracy of over 98% after only ten training epochs.

LCZero and Deep Learning Models

Leela Chess Zero is a free, open-source, and neural network-based chess engine and distributed computing project.  It is a public attempt by multiple developers to achieve what Google DeepMind's AlphaZero was able to accomplish in competitive play against Stockfish 10.

lc0-js is an emscripten port of the lc0 engine, very much like stockfish-js is a port of stockfish to the browser. The lc0 engine runs into the browser, without any server computational resources, without need of any pluggins, and possibly offline. tensorflowjs runs the neural network, and uses the GPU to accelerate the computations.

Disaster Response and Natural Language Processing

This project uses natural language processing (NLP) and supervised machine learning to classify social media messages for disaster events into 36 categories based on tweets data. Included is a command line application and a web dashboard where users can input new tweet and view classification results.
